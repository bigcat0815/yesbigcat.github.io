Unity Skill과 Effect를 분리하는 이점 04

역할의 분리의 이점

  1. 코드의 모듈화(컴포넌트) 및 재사용성 증가

  2. 개발 효율성 향상

  3. 확장성 및 유지 보수성 개선

  4. 게임 밸런싱에 유리

코드의 모듈화(컴포넌트) 및 재사용성 증가

  1. 모듈화: 유지 보수의 용이성

    - 이점1 : 특정 모듈에 변경 사항이 있을 때, 해당 모듈만 수정

    - 이점2 : 이펙트 시스템에 버그가 발생했을 경우, 이펙트 모듈만 집중적으로 디버깅

    - 이점3 : 투사체의 이동 속도를 조정하려면 투사체 모듈만 수정

    - 상황: 게임에서 다양한 스킬이 존재하며, 각 스킬은 서로 다른 효과와 동작

    - 스킬 모듈: 스킬 사용 조건, 쿨다운 관리, 자원 소모 등을 처리

    - 이펙트 모듈: 능력치 변화, 상태 이상 등 캐릭터에 적용되는 효과를 처리

    - 투사체 모듈: 투사체의 이동, 충돌, 생명 주기 등을 처리

  2. '이동 속도 감소' 이펙트의 재사용 예시

    - 스킬 A: 얼음 화살 스킬은 적에게 피해를 주고 이동 속도를 감소

    - 스킬 B: 얼음 폭풍 스킬은 범위 내의 모든 적에게 이동 속도 감소 이펙트를 적용

    - 스킬 C: 함정 스킬은 적이 함정에 걸렸을 때 이동 속도를 감소

    - 이점1 : '이동 속도 감소' 이펙트를 여러 스킬에서 공유하여 사용

    - 이점2 : 이펙트의 수치나 지속 시간을 변경해야 할 때, 한 곳에서 수정하면 모든 스킬에 적용

  3. 투사체의 재사용 예시

    - 화살 투사체: 궁수의 일반 공격, 독 화살 스킬, 연속 사격 스킬에서 모두 동일한 화살 투사체를 사용

    - 마법 탄환 투사체: 마법사의 여러 스킬에서 같은 마법 탄환 모양의 투사체를 활용

    - 이점1 : 동일한 투사체를 사용함으로써 개발 시간과 자원을 절약

    - 이점2 : 플레이어에게 익숙한 시각적 요소를 제공하여 게임 경험을 향상
