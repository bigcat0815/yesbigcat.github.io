서버와 클라이언트 SSH 설치

Window 10 이상에서는 OpenSSH 클라이언트가 설치되어 있다.

1. 윈도우(클라이언트)에서 ssh 키 생성

  1. ssh-keygen

  2. 아래 메세지는 접속시 경로와 비밀번호를 설정을 Enter 눌러 Skip 하자

    Generating public/private rsa key pair. Enter file in which to save the key (C:\Users\사용자이름/.ssh/id_rsa): Enter passphrase (empty for no passphrase): Enter same passphrase again:

  3. id_rsa (개인 키)와 id_rsa.pub (공개 키) 파일이 .ssh 폴더에 생성

    SSH 이름은 다를 수 있으나 .pub 확장자는 공개키이다.

  4. ssh-keygen -t ed25519 -C "bigcat0815@gmail.com"

    email등록 -> ed25519 키 이름이므로 다를수 있다.

2. 공개키 출력 복사

  1. Get-Content 콘솔(파워셀)에서 텍스트를 출력해준다.

    Get-Content C:\Users\사용자이름\.ssh\id_rsa.pub
    cat C:\Users\사용자이름\.ssh\id_rsa.pub

  2. 둘 중에 하나를 사용해 출력된 모든 라인을 복사한다.

3. 공개키 서버에 붙이기

  1. ssh 사용자이름@우분투서버IP -p 22

  2. mkdir -p ~/.ssh 폴더 생성

  3. chmod 700 ~/.ssh권한 부여

  4. echo "복사한_공개_키" >> ~/.ssh/authorized_keys 복사키 추가

  5. chmod 600 ~/.ssh/authorized_keys 인증파일 권한

  6. 터미널을 재시작하면 ssh키로 접속이 된다.

SSH는 클라이언트에서 생성하여 공개키만 서버에 전송

4. github 공개키 서버에 붙이기

  1. GitHub settings | SSH and GPG keys | New SSH key

  2. cat C:\Users\사용자이름\.ssh\id_rsa.pub

  3. Add SSH key 완료를 한다.

  4. 접속 테스트 : ssh -T git@github.com

  5. yes 눌러 진행한다.

  6. 원하는 git저장소 remote 설정

    git remote set-url origin git@github.com:username/repo.git

  7. remote 저장소 확인 : git remote -v

GitHub의 인증은 Https,SSH 방식이 있다.
