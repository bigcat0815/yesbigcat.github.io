Teamcity_Linux_Agent설치

리눅스 TeamCity 서버 설치시 리눅스 Agent 설치되어 있음

1.Java 설치

  • sudo apt update sudo apt upgrade sudo apt install openjdk-17-jdk -y java -version echo "export JAVA_HOME=$(dirname $(dirname $(readlink -f $(which java))))" | sudo tee -a /etc/profile.d/java.sh sudo chmod +x /etc/profile.d/java.sh source /etc/profile.d/java.sh

2. 리눅스 Agent 재설치

  1. 최초 설치라면 다운로드 부터 시작

    Teamcity Server의 Agent install에서 경로를 복사한다

    # 기존 에이전트 디렉토리로 이동(확인) cd /opt/teamcity-agent # 서비스 중지 sudo systemctl stop teamcity-agent # 기존 파일 제거 (백업 필요 시 백업 후 삭제) sudo rm -rf /opt/teamcity-agent/* # 최신 에이전트 다운로드 및 설치 sudo wget http://192.168.0.4:8111/update/buildAgent.zip sudo unzip -d /opt/teamcity-agent buildAgent.zip

  2. teamcity-agent 유저 생성및 소유권 이전

    sudo adduser teamcity-agent sudo chown -R teamcity-agent:teamcity-agent /opt/teamcity-agent

3. 리눅스 Agent 설정

  1. buildAgent.properties 설정

    ownPort는 Agent마다 다르게 설정

    # 에이전트 설정 (server URL 등) sudo nano /opt/teamcity-agent/conf/buildAgent.properties

  2. 수정 내용

    serverUrl=http://192.168.0.4:8111 name=YesAgent ownPort=9091

4. 리눅스 Agent 서비스 등록

  1. teamcity-agent.service 등록

    sudo nano /etc/systemd/system/teamcity-agent.service

  2. 수정 내용

    [Unit] Description=TeamCity Build Agent After=network.target [Service] Type=forking User=teamcity-agent Group=teamcity-agent ExecStart=/opt/teamcity-agent/bin/agent.sh start ExecStop=/opt/teamcity-agent/bin/agent.sh stop Restart=always RestartSec=10 [Install] WantedBy=multi-user.target

5. 서비스 재시작

  1. 재시작

    sudo systemctl daemon-reload sudo systemctl start teamcity-agent sudo systemctl enable teamcity-agent

  2. 서비스 확인

    sudo systemctl status teamcity-agent
