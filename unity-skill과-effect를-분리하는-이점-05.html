Unity Skill과 Effect를 분리하는 이점 05

개발 효율성 향상

  1. 병렬 개발 가능: 팀 작업의 효율성

    - 개발 팀 구성:

    - 스킬 개발자: 스킬의 로직과 사용 조건을 구현

    - 이펙트 개발자: 이펙트의 적용 및 상태 변화를 구현.

    - 투사체 개발자: 투사체의 이동과 충돌 처리를 구현.

    - 시각 효과(VFX) 아티스트: 시각적 효과를 제작.

    - 이점1: 각자가 자신의 영역에 집중하여 동시에 작업하므로 전체 개발 기간이 단축

    - 이점2: 각 분야의 전문가들이 최적의 결과물 도출

  2. '빠른 수정 및 업데이트

    - 상황: 게임 테스트 중 특정 이펙트의 지속 시간이 너무 길어 게임 밸런스에 문제

    - 대응: 이펙트 모듈에서 해당 이펙트의 지속 시간 값을 조정

    - 이점1 : 다른 모듈에 영향 없이 즉시 수정이 반영

    - 이점2 : 문제를 신속하게 수정하여 게임 품질을 유지

    - 이점3 : 다른 시스템에 영향을 주지 않으므로 버그 발생 가능성이 낮음
