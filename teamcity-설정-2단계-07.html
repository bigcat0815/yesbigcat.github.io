TeamCity 설정 2단계 07

7. TeamCity Job 설정 2

  1. Job 1 → step2 → SCRIPT 선택

  2. Step name: Deploy to github-pages

  3. Working directory: bin/ 에서 result를 찾아서 githubpage에 배포함

  4. script는 아래 기술

Step2 script

  1. GIT_TOKEN → 다음 단계에 설명

  2. GIT_USER → 계정 ID

  3. GIT_REPO → @github.com/계정/githubpage프로젝트의 git 주소

  4. cp -R ../result/* . 작업 폴더위치가 안맞으면 빌드 fail이 된다.

  5. Working directory: bin/ 에서 result를 찾아서 githubpage에 배포함

  6. 아래 스크립트 : step2 script

step2 script
#!/bin/bash set -e # 환경 변수에서 GitHub 토큰 가져오기 GIT_TOKEN=%env.GIT_TOKEN% GIT_USER=bigcat0815 GIT_EMAIL=bigcat0815@gmail.com GIT_REPO=https://$GIT_USER:$GIT_TOKEN@github.com/bigcat0815/bigcat0815.github.io.git TARGET_BRANCH=main # Git 설정 git config --global user.name "$GIT_USER" git config --global user.email "$GIT_EMAIL" # main 브랜치 클론 git clone --branch $TARGET_BRANCH --single-branch $GIT_REPO site cd site # 기존 파일 삭제 git rm -rf . # 압축 해제된 파일 복사 cp -R ../result/* . # 변경 사항 커밋 git add . # 변경 사항이 있는지 확인 if ! git diff --cached --exit-code > /dev/null; then # 변경 사항이 있을 경우 커밋 및 푸시 git commit -m "Deploy to GitHub Pages via TeamCity build" git push origin $TARGET_BRANCH echo "Deployment successful." else # 변경 사항이 없을 경우 메시지 출력 echo "No changes to deploy." fi
