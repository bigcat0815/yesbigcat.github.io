Unity Skill과 Effect를 분리하는 이점 06

확장성 및 유지 보수성 개선

  1. 새로운 기능 추가 용이

    - 새로운 스킬 추가: '전기 충격' 스킬을 추가

    - 스킬 개발자는 기존의 스킬 클래스를 상속하여 새로운 스킬 클래스 생성

    - 이펙트 개발자는 '마비' 이펙트를 생성하여 대상의 행동을 제한하는 로직 구현

    - 투사체가 필요 없다면 투사체 모듈을 건드릴 필요가 없음

    - 이점1: 이미 구축된 스킬, 이펙트 시스템을 활용하여 빠르게 새로운 스킬을 추가

    - 이점2: 코드를 필요한 부분만 구현하고 공통 기능은 재사용

  2. '버그 발생 시 디버깅 용이

    - 상황: 플레이어가 '독 화살' 스킬을 사용할 때 게임이 크래시

    - 대응1: 문제의 원인을 찾기 위해 스킬 모듈, 이펙트 모듈, 투사체 모듈을 각각 검사

    - 대응2: 로그를 통해 투사체 모듈에서 예외가 발생한 것을 확인

    - 대응3: 투사체의 충돌 처리 로직에서 null 참조 에러가 있음을 발견하고 수정

    - 이점1 : 모듈이 분리되어 있으므로 어디에서 문제가 발생했는지 빠르게 파악

    - 이점2 : 특정 모듈에 집중하여 문제를 해결하므로 시간과 노력이 절약
